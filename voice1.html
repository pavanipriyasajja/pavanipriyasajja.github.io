Voice User Interface mobile application for Thermostat

OVER VIEW OF THE PROJECT

Thermostat company would like to rework on their existing theromost to improve their existing features and introduce a new voice feature mobile application for their thermostat so the user can easily use their thermostat.

METHODS

User Research, User interviews, User personas, Storyboard, Dialog flow, User flows, Low fidelity prototype, High fidelity prototype. In person User test.

TOOLS

Adobe XD, Pen & Paper, Adobe Illustrator, Adobe Photoshop. Google documents.

CLIENT

This is one of a Bellevue college UX design course project. This is a solo project.

TIMELINE

4 Months (Oct 2019 - Jan 2020).

STATUS

Completed

MY ROLE

Taking help from the 3 other design students, I have conducted user research to identify problem context areas and find out key insights to design mobile applications for the thermostat with voice user interface.

Created multiple interactions to work together since the voice commands and the user interface of the application work at same time. Designed and developed high fidelity clickable prototypes using the Adobe XD tool and conducted In person user test with 3 participants on the prototype to implement the concept.

DESIGN PROCESS

DESIGN CHALLENGE

Establish the concept of the Voice User Interface mobile application for the thermostat since the user can use the application from anywhere and anytime.

Create multiple flows like the user interface of the application as well as the voice commands of the application work together since the user can easily operate the thermostat.

Increase the amount of users to use the VUI mobile application for the thermostat by reducing manual use of the thermostat.

SOLUTION

OUTCOMES

Designed and developed a clickable high fidelity prototype to establish the concept of Voice User interface mobile application for the Thermostat.

Created basic thermostat operation flows for the user to operate the thermostat by giving voice commands.

Created scheduling mode and holiday mode user flows for the thermostat so the user can schedule their thermostat as per their desired day and time and temperature.

PROCESS

UNDERSTAND PROBLEM CONTEXT AREA

Storyboarding

I have come up with user storyboarding for the thermostat to express the ideas of how the user having trouble operating manually with their thermostat and if the user have a mobile application with the VUI features that reduce their effort of operating it.

User Research & Research Key Insights

I have started to understand problem context areas by conducting user interviews with the 3 different participants of familiar with voice thermostat, absolute new to the voice commands and finding key insights as well as comeup with business perspective research key insights.

User Research Key Insights (User Perspective)

  • Users work hard to operate the existing thermostat Manually.
  • Users would like more features for their home operations.
  • Thermostat have interactive mobile Interface with the Voice Commands since they can use the thermostats easily.

User Research Key Insights (Business Perspective)

  • Create an app that has rich voice commands
  • Increase the number of new users to use voice controlled app
  • Functionality of the app needs to be more effective and intuitive while using.

Create multiple user flows for the application

Gathering ideas from the User research key insights, I drafted the user flow(Dialog flow) that the user would attempt to achieve their goals. For the voice user interface applications the app needs to work on two different interactions together. Since we created a dialog flow which helps us to design and integrate a conversational user interface into mobile apps. We created three various dialog flows for three different kinds of target users.

User flows help us on mapping out each and every step the user takes—from entry point right through to the final interaction in our voice User Interface of thermostat application.

USER TESTING & KEY INSIGHTS

I have conducted In person Usability studies with 3 participants to identify the user problems while user interacting with our thermostat and we found out important key insights.

User tesing key insights from first iteration of the application.

User felt priority option information but when the user wanted to change the priority room from activate to non activate user did not find it. Since then, I have edited my prototype with priority settings which makes it easy to understand the activated and unactivated room.

Users got confused or bored when scheduling a new mode. When the user comes to scheduling for the date, day and night mode the user feels it’s taking too much time and a couple of things are repeating as well since the user thinks if there is a default option then the user can schedule quickly.

Some voice commands take much time to produce results. Animation of the app should be a little fast so that users can navigate the app quickly.
